Why you should use BEM

There are plenty of methodologies out there which aimed on reducing CSS codebase and organizing programmers cooperation and maintaining of CSS code. This is obvious in large projects like Twitter, Facebook and Github, but other projects often grows into same "Huge CSS file" state pretty quickly.

Here are some reasons, that can convince you to use BEM or other methodology in your front-end:

  • Performance — smaller number of CSS selectors (because of reusing Blocks) causing better performance and experience of site.
  • Productivity — clear and structured CSS is easier to maintain and develop.
